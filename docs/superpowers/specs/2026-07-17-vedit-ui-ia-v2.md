# vedit UI 情報設計 v2 — 「自己説明する文法」仕様

日付: 2026-07-17。入力: Fable 根本レビュー(docs/ui-legibility-review-fable.md
S1〜S9)+ Codex 根本レビュー(P1×4, P2×3, P3×1 + 再構成案・用語表・空状態)。
動機(ユーザー): 「アプリのデザインが根本的にわかりにくい。商用アプリに
比べるとどうしても」。

## テーゼ

現画面には「相棒の確認画面」「簡易NLE」「システム管理画面」の3つの
メンタルモデルが同時に出ている。機能不足ではなく**役割分担が画面構造に
現れていない**ことが商用アプリとの差。修正原則は商用 NLE の予測可能な
役割分担 — 素材は素材庫、出力はプレビュー、時間構造はタイムライン、
選択対象の変更はインスペクタ — を、相棒モデル(操作の主体は Claude、
UI は確認・承認・微調整)を壊さずに借りること。

## 画面の4領域(固定)

1. **上部: プロジェクトと相棒状態** — プロジェクト名・現在タイムコード・
   出力比率+**Claude 状態の常設表示**(`Claude: 待機中 / 編集中 /
   あなたの確認待ち` — WS 活動と保留候補数から導出、新 API 不要)。
   「現在の版 N」は右端の補助情報に格下げ。
2. **中央: プレビュー** — 現状維持。show 指示の候補/比較カードは
   この製品固有の強みなので残す。ただし出所ラベル「Claude の提案」を付す。
3. **下部: ラベル付きタイムライン** — 左端に動的トラックラベル
   (`V2 B-roll / V1 本編 / T1 テロップ / A1 音声 / A2 BGM` — 存在する行だけ)、
   上部に簡素な時間目盛り。選択中クリップはハイライト。
4. **右側: 仕事別パネル** — 常設タブ `確認 / 素材 / 文字起こし / 履歴`。
   選択発生時はタブ列を増やさず、右ペインが一時的に**インスペクタ**へ遷移
   (上部に `← 確認に戻る`)。

## 確認タブ(旧「いま」)

- 2群に分離: 上段「**Claude の編集提案**」(無音・フィラー等の候補、
  カットする/残す)、下段「**対応が必要**」(色・アンカー切れ・低信頼
  文字起こし・QC 警告)。
- バッジの整合: タブバッジは2群の合計だが、群ごとの見出しに内訳。
  一括ボタンは**グループヘッダの1箇所だけ**、文言は結果明示
  「無音候補2件をカット(−2.4秒)」。最上部の「すべて承認」は廃止。
- 無音検出のつまみは候補グループ内の折りたたみ「検出条件」へ。
  名称「無音として候補にする最短時間」(実装は minGap=秒 — 現行の
  「しきい値」は不正確)、ボタン「この条件で候補を作り直す」。
- 候補カード: 日本語テンプレート「"かった"の後の無音 0.6秒 — 詰めると
  −0.3秒」。カードクリック ⇔ タイムライン該当区間ハイライトの相互リンク。
- 変更履歴は「履歴」タブへ移設(人間語要約は現行実装を流用)。

## インスペクタ(選択時の右ペイン)

- クリップ: 素材別名・タイムライン位置・使用範囲・±1F トリム・クロップ
  状態・「タイムラインから外す」。
- 字幕 cue / B-roll / モーション / スプライト / セリフも同じ位置に対象別
  表示(既存のポップオーバー/細バーの機能を移植・集約)。
- **字幕の常設入口**(2026-07-17 ユーザー質問「文字の編集とかデザインとか
  変えられないの?」で入口不可視が実証された): プレビュー上の cue クリック
  だけでなく、タイムラインの T1 行クリック/確認タブの「字幕のデザインを
  変える」入口からスタイルインスペクタ(既存 W-CAP ポップオーバーの内容:
  本文・フォント[キット+OS]・文字/縁/箱色・サイズ・縁太さ・背景不透明度・
  縦位置ドラッグ)に到達できること。
- **音声パネル**(同質問「音声周りの設定とか簡単にできないの?」— 現状
  UI に音声操作が皆無): A2/BGM 行の選択→インスペクタで 音量(gain)/
  フェードイン・アウト/ダッキング on-off(既存 music-update op)、
  BGM 追加ボタン(music-add op、--sfx 相当のトグル)、プロジェクト全体の
  「会話リペア」プリセット選択 outdoor/indoor/wireless/off(audio-repair op)。
  クリップ単位の gain/mute はモデル拡張が必要なので検証後の機能波
  (ロードマップ v3 掲載済み)。
- 露出しすぎない: キーフレームや詳細 EQ 等は出さない(会話側との責任境界)。

## 用語統一(全 UI 文字列に適用)

| 現在 | 提案 |
|---|---|
| いま | 確認 |
| 要確認 | 確認待ち |
| 候補 | Claude の編集提案 |
| すべて承認 | (廃止 — グループの「候補N件をカット(−X秒)」に統一) |
| まとめて承認 / まとめて却下 | まとめてカット / まとめて残す |
| 承認 / 却下 | カットする / 残す |
| Transcript | 文字起こし |
| keep / reject / 未確認 | 採用 / 不採用 / 未確認 |
| keepだけで仮タイムライン作成 | 採用シーンで仮編集を作る(採用>0 のときだけ表示、「(戻せます)」明記) |
| 変更 #7 | 現在の版 7(右端・弱表示) |
| 変更履歴 | 編集履歴 |
| この直前に戻す | この編集より前に戻す |
| 無音のしきい値 | 無音として候補にする最短時間 |
| 再検出 | この条件で候補を作り直す |
| クリップを外す | タイムラインから外す |
| アンカー切れ | 配置先を見失った B-roll/画像 |
| 低信頼のトランスクリプト | 聞き取りを確認したい箇所 |
| IN / OUT | 開始点 / 終了点(IN/OUT) |

- **CLI 構文の UI 露出を全廃**: 「`vedit transcribe` を実行」→
  「Claude に『この素材を文字起こしして』と伝えてください」。手動で直す
  導線は「手動で微調整」、会話へ戻す導線は「Claude に伝える」で統一。
- 説明の文字壁(Transcript 冒頭・素材タブのキーボード凡例)は全廃し
  「?」オーバーレイへ集約。

## 空状態

- プロジェクト空: 中央に「最初の動画を追加 / ここに動画をドロップするか、
  Claude に『この動画を編集して』と伝えてください / 元動画は変更しません」。
  右ペインは「確認するものはまだありません」のみ。
- 文字起こし未実行: 任意である理由+素材ごとの 未実行/処理中/完了+
  Claude への頼み方。
- 候補ゼロは4状態を区別: 未検出「編集提案はまだ作られていません」/
  検出中「Claude が無音や言い直しを確認しています」/ 全確認済み /
  問題なし。ゼロ時は一括ボタンと検出条件を隠す。

## やらないこと(商用から借りない)

パネルドッキング/ワークスペース、常設ツールパレット(Razor 等)、
多数の空トラック、全パラメータ・インスペクタ、深いビン階層、
**UI 内の第二 AI チャット**(主導権が二つになる)、タイムライン先行の
オンボーディング(教えるのは「頼む→光る→確認→戻せる」の一巡)。

## 波2 追加項目(2026-07-17 到達可能性・parity 総点検より)

- **字幕のデッドロック解消**: captions.enabled の常設トグル+プリセット
  (style)選択+maxChars をスタイルインスペクタに追加(enabled=false だと
  cue が消えて字幕 UI への入口ごと消滅する現行構造の解消)。
- **音声パネル拡張**: audio-mix(目標ラウドネス・duckAmount)も対象に
  (duckAmount はプレビューが常時参照しているのに変更手段がない)。
- **add/remove の対称化**: music-add の追加ボタン、選択インスペクタに
  削除(motion/music/broll)。**motion-add はボタンを作らない**
  (Codex 判定を採用: claude-only-with-guidance — 「頼むチップ」で誘導)。
  sprite/dialogue/compose 系の追加 UI もコンポジション UI 波(波3)へ。
- **color-adjust のプレビュー近似**: exposure→CSS brightness、
  saturation→saturate で近似反映。WB は「書き出しで確認」の注記を表示
  (嘘の近似はしない)。
- **セリフの通常プロジェクト・プレビュー描画**(tick から
  renderDialogueBubbles を呼ぶ — レンダー側は既定焼き込みに修正済み)。
- **再生ボタンの嘘を直す**: セグメント0のとき pressed 化しない+
  disabled+理由 title。
- **保護区間行**: クリックでシーク+「Claude が守っている無音」の title。
- **QC kit-asset-missing** を確認タブ「対応が必要」群に追加(現在
  どの経路でも不可視)。
- **「言い直しを確認」ボタン**(GET /api/takes → 既存 takesCard を
  ユーザー起点で開く — 現在は Claude の show directive 経由のみ)。
- **キットのリンク状態表示**(素材タブ先頭に「キット: 名前」+スタイル/
  アセット数。未リンクなら非表示)。
- **自動保存の明示**: ヘッダーの版表示に「自動保存」を添える
  (「保存」という語が UI に一度も出ない現状の解消)。
- **波3確定(Codex 推奨採用): 書き出し結果カード** — 実行 UI は作らない。
  CLI が書き出し完了時に {ts, kind, file, ok, revision, options, warnings,
  captionsBurned…} を cache/export-results.json に原子的追記(直近20件)、
  daemon は読み取り専用 `GET /api/export-results` のみ、web は確認タブに
  「最後の書き出し」カード(revision が現在と違えば「古い版 — 再書き出しを
  Claude に頼んでください」警告、custom-html 警告等もここに残る)。
  空状態は「Claude に『MP4 を書き出して』と伝えてください」。
  コンポジション編集 UI は引き続き別枠(claude-only-with-guidance)。

## 実装ノート

- 既存資産を移設で使う: #clipInspector(index.html:93)、#claudeStrip
  (:115)、humanizeRevision、sourceLabel(パンチリストで導入)、
  show ディレクティブ(:43)。
- トラックラベルは絶対配置行(style.css:316 の 92px 座標系)の左に
  固定ガター列を追加する構造変更。行の表示/非表示は既存データ有無で。
- Claude 状態: WS メッセージ受信でアクティブ化する既存 claudeStrip の
  ロジックを常設ヘッダー表示に昇格(候補>0 → 確認待ち、show/edit 受信中
  → 編集中、他 → 待機中)。
- 規模: 領域再編+インスペクタ+トラックラベル=L、用語・空状態=S〜M。
  2波に分割可(波1: 用語+確認タブ分離+空状態+検出条件格納、
  波2: インスペクタ+トラックラベル+Claude 状態)。

## 受け入れ基準

初見ユーザーが予備知識ゼロで: (1) どの帯が何のトラックか言える、
(2) クリップを選ぶと何が起きたか見え、そこで何ができるか分かる、
(3) 「確認待ち N」の N が何を指すか説明できる、(4) 困ったとき
「Claude に頼む」へ導かれる、(5) UI のどこにも英日混在文・内部 ID・
CLI 構文が出ない。
