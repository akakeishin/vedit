# N2 デザイン波「計器盤」— frontend-design ブリーフ

日付: 2026-07-18。起草: ディレクター(Fable、frontend-design スキル準拠)。
入力: 現行 UI スクショ実査+andashi 記録(docs/andashi-2026-07-18.md、K6/K7 採用)。
実装: Sonnet サブエージェント。対象: web/index.html, web/style.css, web/app.js
(+K6 用の daemon 読み取り専用エンドポイント1本のみ)。

## 主題の把握(design lead としての判断)

vedit の世界は「編集室」。この製品固有の状況は、**操作の主体が相棒(Claude)で、
人間は確認と知覚チューニングをする**こと。商用 NLE の借用(W-DESIGN)で構造は
できたが、現状のヘッダーは等価な文字列の羅列で、最頻参照の2情報 —
「今どこにいるか(時間)」「相棒はいま何をしているか」— が埋没している
(スクショ実査で確認)。今回の一手はこの2つを**読む情報から見える計器**に変える。

トークン(#131315 系・IBM Plex・白プライマリ・3カラム)は Claude Design
ハンドオフの正を**変更しない**。これはハンドオフの上に載る「質の実行」波。

## 自己レビュー(generic 回避)

- 近黒+単アクセントは AI 定番だが、これはユーザー承認済みハンドオフの指定 —
  ブリーフの言葉が勝つ。自由裁量部分を定番に使わないこと: シグネチャは
  色ではなく**計器の振る舞い**(タイポグラフィ階層+状態のランプ文法+
  押している間だけの時間跳躍)に置く。
- フィルムのスプロケット穴モチーフは検討の末**却下**(アクセサリー2個目。
  Chanel の規律: 外す)。装飾は足さない。
- 番号付きマーカー等の構造装飾は不使用(内容が順序を持たない)。

## 実装項目

### 1. シグネチャ: トランスポート計器列(中央カラム、プレビュー直下)

現在の `0:00.0 / 0:14.6 [⏮][▶][⏭] f0 / f439 · 30fps` の行を計器列に再設計:

- **現在タイムコードを主役に**: IBM Plex Mono、現行の約2倍(20〜22px)、
  font-variant-numeric: tabular-nums、白(--text-1)。フレーム部(`.4`等の
  下位桁)は --text-3 で一段弱く — 秒までが判断に使う情報、下位桁は精密作業用。
- 総尺 `/ 0:14.6` は現行サイズのまま --text-3。
- フレームカウンタ `f0 / f439 · 30fps` は右端・現状維持(既に正しい弱さ)。
- ヘッダー(上部バー)の `0:00.0` 重複表示は**削除**(計器列に一本化。
  ヘッダーは版・自動保存・出力仕様のプロジェクト情報のみに)。
- 再生中はタイムコードが毎フレーム更新(既存 tick)。シークは既存の同期描画
  (renderPlaybackFrame)を必ず経由 — ここは回帰実績のある経路、壊さない。

### 2. シグネチャ: タリーランプ(Claude 状態)

ヘッダーの「Claude・1件 — 確認待ち」ピルを**ランプ+短文**の文法に置換:

- ランプは 8px 円 + 1px 外輪。状態文法:
  - 待機中: 外輪のみ(消灯)。文言「Claude: 待機中」
  - 編集中: 白点灯+ゆっくり呼吸(opacity 0.55→1.0、周期 2.4s、
    `prefers-reduced-motion: reduce` では点灯固定)
  - 確認待ち: --pending 琥珀で点灯(アニメなし)+件数「確認待ち 1件」
- 意味色規律は不変: 琥珀=確認待ちのみ、--ok 緑=接続のみ。白点灯は
  無彩色なので規律違反にならない。**グラデーション・グロー装飾は禁止**。
  box-shadow によるプレビュー枠エッジ発光は検討の末**却下**(装飾化リスク、
  reduced-motion 分岐の保守が割に合わない — Challenger 指摘の採用)。
- 状態導出は既存 claudeStrip のロジックを流用(候補>0→確認待ち、
  show/edit 受信中→編集中、他→待機中)。

### 3. K6「押している間、直前」(機能+インタラクション)

1手適用直後の「効いた?」を1操作で返す、この波の機能的シグネチャ。

- **UI**: 計器列の右寄りに押下ホールド型ボタン「直前と比べる」。
  pointerdown で直前 revision の見えへ切替、pointerup/leave で現行へ復帰。
  キーボード: フォーカス+Space/Enter 押下中も同挙動(keydown/keyup)。
  ホールド中は計器列に「直前の版 N-1 を表示中」を --pending ではなく
  --text-2 で表示(警告ではない、状態表示)。プレビュー左上に小ラベル
  「直前」を重ねる(押している間のみ)。
- **データ**: daemon に読み取り専用 `GET /api/manifest-at?revision=<n>` を追加
  (revisions.jsonl から該当版の manifest を再構成して返す。書き込みなし)。
  web は現行 revision-1 を初回ホールド時に取得しキャッシュ(revision が
  進んだら破棄)。
- **描画**: ホールド中は保持した直前 manifest でタイムライン→ソースの
  マッピングを再計算し、**同一プレイヘッド時刻**のフレームを既存の同期描画
  経路で表示。字幕 cue・オーバーレイも直前 manifest 基準で描く。音声は
  切り替えない(映像のみの知覚比較)。タイムライン帯の描画は現行のまま
  (プレビューだけが過去を見る)。
- **境界**: revision<=1 ではボタン disabled+title「まだ比較する版がありません」。
  undo 直後は「直前」= undo 前の版になる(revision 連番に忠実で良い)。
  適用直後でなくても常時使えて良い(直前1世代限定)。
- **禁止**: 任意版ピッカーは作らない(構造判断への越境。andashi 記録参照)。

### 4. 直し: 素材カードのバッジ折返し(task chip 由来)

.srcRow のバッジ(再生/タイムラインへ追加/詳細)が縦に折り返して
カードが間延びする既存バグを修正。カード下部で横一列・省スペースに収める
(アイコン+短ラベル、カード幅 300px 内で収まる寸法に)。

### 5. ロードの所作(ひとつだけの演出)

プロジェクト読み込み完了時、トラック行を下から順に 60ms 間隔で
フェード+4px 上昇(計 250ms 以内、1回だけ)。`prefers-reduced-motion:
reduce` では無効。再接続・revision 更新では発火しない(初回ロードのみ)。
これ以外のページ内アニメーションは追加しない。

### 6. 小さな秩序(ついで修正、見つけた場合のみ)

- テロップ帯の色が意味色規律外の青系に見える場合は無彩色系
  (--surface 系の明度差)へ寄せる。
- オーディオ節のスライダー(ダッキング量・ラウドネス)のラベル・値の
  タイポグラフィを計器列と同じ tabular-nums に揃える。

## 受け入れ基準

1. 5秒見れば「今どこ・Claude が何をしているか」が読まずに分かる
   (タイムコード階層+ランプ)。
2. 「直前と比べる」を押している間だけ直前版の映像が同一プレイヘッドで
   表示され、離すと即座に現行へ戻る。マウス・キーボード両対応。
3. `prefers-reduced-motion: reduce` で呼吸・ロード演出が消えても
   情報が欠けない。
4. 意味色規律違反ゼロ(琥珀=確認待ちのみ、緑=接続のみ、装飾グラデなし)。
5. 既存 DOM id はすべて保持。`node --check web/app.js` 通過、
   コンソールエラーゼロ、既存 e2e(test:e2e)green。
6. 素材カードのバッジが 300px 幅で横一列に収まる。
