# vedit IA v3「調整室とキューシート」統合仕様

日付: 2026-07-18。起草: ディレクター(Fable)。
入力: andashi 構造波(Scout 8案+直列1案)/意匠波(D1-D3+D3v2)/
Opus 反証2本(意匠=条件付き支持、構造=背骨は覆すべき+救済節)/
ユーザー Commit 2回(合成案 → 反証を受けた修正統合案)。
関連: docs/andashi-2026-07-18.md、specs/2026-07-18-vedit-design-wave-n2.md(実装済み)、
specs/2026-07-18-vedit-overlay-stack.md(src 実装中)。

## 0. 決定の要約

- **構造**: 常設ワークスペース(3カラム+下段タイムライン)は**維持**。
  「フィード=ホーム」は構造反証により棄却(在席ワークフローで lastSeen が
  無意味化/知覚チューニングが常設プレビューを要求/常設面の入口を失う)。
  採るのは救済節の3機構: キューシート化・時刻同期スリムバー・ドリルイン文法。
- **意匠**: 二面構成「調整室とキューシート」— 部屋(左+中央+下段)=
  D3v2 の暗い計器、右パネル= D2 由来の紙。D1 の暖光は紙と確認待ちを照らす
  ランプとして限定使用。視覚の正= scratchpad design-directions/d4-cuesheet.html
  (ゲート後に web/ へ写す。スクショを docs/design-refs/ に保存すること)。
- **フォント**: システムゴシック+ui-monospace(ユーザー指示。メモリ固定済み)。
- **言語**: UI 文字列を辞書化し ja/en 切替(サーバー生成文は当面 ja)。
- **コピー**: 日本語全文言をディレクターが推敲(§4 の規範+対照表)。

## 1. 構造仕様

### 1.1 右パネル=キューシート(現行 claudeView の進化、置換ではない)

- 現行の2群分離(「Claude の編集提案」承認群/「対応が必要」警告群)は**維持**。
  フィードへの再フラット化はしない(反証2の採用)。
- その上に**新着の直交軸**を重ねる:
  - localStorage に `lastSeenRevision[projectDir]`。更新タイミングは
    「キューシートが可視のままユーザー操作(クリック/キー)が発生した時点の
    現 revision」(単に開いただけでは進めない — ぼんやり開きっぱなし対策)。
  - それより新しい版に由来する項目に「新着」マーク(朱/琥珀ドット)+
    パネル先頭に区切り行「― 前回の確認から N件 ―」。
  - **未決保護**: 未承認の提案は lastSeen がいくつ進んでも常に表示・
    常に承認可能(既読≠既決。反証2の「決定の喪失」対策)。
- 「実行済み」= revision ログ由来の完了項目(相棒が実行した操作の人間語
  要約、行為者つき)を新着区間に限り表示。全履歴は従来どおり履歴タブ。
- **紙を白紙にしない**(2026-07-18 ユーザー指摘「右ペインが何もないアプリも
  珍しいね」への構造的回答): 提案ゼロ・新着ゼロの定常状態でも、キューシートには
  常に (1) 直近の作業記録 3〜5件(新着でなくても表示、弱トーン)、
  (2) 最後の書き出し結果のミニカード(あれば。版が古ければ警告つき)、
  (3) プロジェクトメモ(vedit note の policy/todo 先頭数件、あれば)、
  (4) 「次の一手」チップ 1〜2枚(状態から導出: 転写未実行なら
  「文字起こしを頼む」、BGM なしなら「BGM を頼む」等の Cowork コピーチップ)
  が載る。受信箱ではなく「机の上の記録紙」— 片付いても白紙にならない。
- 空状態(新着ゼロ・提案ゼロ)の見出し文言は現行の4状態を維持しつつ、
  上記の常設内容の上に短く表示する。

### 1.2 時刻同期スリムバー(A-1 の常設ワークスペース内実装)

- プレビュー下・計器列の上に**高さ固定の専用行**(~44px)を常設予約
  (リフローなし・transport を覆わない — 反証1の採用)。
- 再生中、プレイヘッドが提案区間の 2 秒前に入ると予告
  「まもなく: "かった"の後の無音 0.6秒」、区間通過中は
  [カットする][残す] を表示。通過後 1 秒で消灯。停止中は空(薄い罫のみ)。
- キューシートの該当カードと同一データ・同一状態(片方で承認すれば
  両方から消える。入口は2つでも状態は1つ)。

### 1.3 ドリルイン文法の統一(F-2 の右パネル内実装)

- 既存 inspectorView / captionView / exportView を「対象スコープ+
  適用して戻る」の**シート文法**に統一(ヘッダ: ← 戻る(文脈名)/
  対象ラベル/適用状態)。常設プレビューの隣で編集する現行原則は不変。
- プロジェクト全体設定(オーディオ・キット・素材)は**左パネル常設のまま**
  (F-2 の未決「全体設定の置き場」は「常設面は左」で解消)。

### 1.4 到達可能性の不変条件

素材一覧・文字起こし全文・タイムライン直接操作・音声パネル・キット状態・
書き出し結果ビューの入口は**一切移動しない**(監査 docs/ui-reachability-audit.md
が閉じた傷を開けない)。本仕様で変わるのは右パネルの中身と中央の1行のみ。

## 2. 意匠仕様(d4-cuesheet.html を正とする)

- 部屋(暗): D3v2 トークン — #0F1013 系、ヘアライン、銘板、読み出し窓、
  コマ番号、タリーランプ(N2 実装済みを継承)。
- キューシート(紙): #F5EFE3 系の紙面、インク文字、印刷罫、朱=
  確認待ち/赤入れのみ。紙上の書体もシステムゴシック。
- ランプ光: 紙パネルへの淡い暖色は d4 のゲート結果に従う(過剰なら無し)。
- 意味色: 暗部=琥珀(確認待ち)/緑(接続)。紙部=朱(確認待ち)。
  それ以外の彩色は既存トラック識別色のみ(反証済みの語彙として維持)。
- reduced-motion: 呼吸・演出は全て静止代替(N2 実装済みの規律を継承)。

## 3. i18n 仕様

- `web/strings.js`(新規): `t(key, vars?)`+ ja/en 辞書。宣言的 DOM
  (data-i18n 属性)+動的文字列の両対応。言語設定は localStorage、
  切替 UI はヘッダー右端の小トグル(ja/EN)。
- 対象: web の全 UI クローム文字列。**対象外(明記)**: revision 要約・
  警告文などサーバー生成で保存される文章(当面 ja。将来は daemon が
  структured データを返し web 側で組み立てる改修が必要 — バックログ)。
- e2e はキー基準(data-testid / id)で書かれているため言語切替に耐える。
  en 状態のスモーク1本を追加。

## 4. 日本語コピー規範(ディレクター起草)

原則:
1. 慇懃語・翻訳調の禁止: 「ご提案」→「提案」、「〜されました」受動態を
   避け「字幕を5件付けました(Claude)」の行為者明示。
2. ボタンは結果が見える動詞: 「カットする/残す/適用して戻る/書き出しを頼む」。
   同じ操作は全画面で同じ語(カットする、を統一語とする)。
3. 状態は名詞+数: 「確認待ち 1件」「新着 3件」。装飾形容詞なし。
4. システム都合語の禁止: セッション、ディレクティブ、レンダー(→書き出し)、
   プロキシ(→プレビュー用データ)等を UI に出さない。
5. 空状態は次の行動を指す: 「確認するものはまだありません。編集は
   Cowork の会話から頼めます」。
6. 相棒の一人称運用: Claude の行為は「Claude が〜しました」、
   ユーザーへの依頼導線は「Claude に伝えてください」で統一
   (Cowork/Claude の呼び分けは「Cowork を開く」ボタンのみ製品名)。

主要対照(実装時に全辞書を抽出し、ディレクターが ja を全数レビューする):
| 現行 | 改訂 |
|---|---|
| キット: 未連携 | キット: なし |
| Cowork 接続中・確認待ち 1件 | Claude: 確認待ち 1件(タリー併記) |
| COWORK セッション | (見出し廃止 — キューシートの日付見出しに吸収) |
| 聞き取りを確認したい箇所: … | 聞き取りに自信がない箇所: …(3語) |
| 検出条件 | 無音とみなす条件 |
| この条件で候補を作り直す | この条件で探し直す |
| 書き出しで確認 | 書き出しで反映されます |

en 辞書は同規範の英語版(Claude 表記維持、Sentence case、動詞ボタン)。
ディレクターが最終レビュー。

## 5. タイムラインのオーバーレイ行

- overlay-stack(実装中)の layer を「重ね L2」「重ね L3」行として、
  **存在するときだけ**表示(既存 B-roll 行の流儀)。
- rect/opacity の調整は右パネルのドリルインシート(数値+9位置プリセット
  +不透明度スライダー)。ドラッグ配置は作らない(装飾オーバーレイの線引き)。

## 6. 実装波と検証

- **波A 構造**: §1.1-1.3(web/app.js・index.html・style.css+daemon 変更なし
  想定。lastSeen は localStorage のみ)。e2e にキューシート新着・スリムバー・
  シート文法のテスト追加。
- **波B 意匠**: d4 トークンの全面適用(style.css 主体)。ビフォー/アフターの
  スクショ照合を波内で実施。
- **波C i18n+コピー**: 辞書抽出→ディレクター ja レビュー→en→切替 UI。
- **波D オーバーレイUI**: §5(src 側の N6 が着地してから)。
- 順序: A→B は直列(同ファイル)。C は B の後。D は N6 依存。
- 各波後に `npm run build`+vitest+e2e green。全波後に
  ミニ検証(シナリオ3系の再走+新機構の実操作)+Codex ゲート+DS 再同期。

## 7. 受け入れ基準

1. 開いた瞬間に「前回から何が起きたか」がキューシート先頭で分かり、
   未決の提案は何があっても消えない。
2. 再生中、提案区間で視線を動かさず承認できる(スリムバー)。
3. 右パネルのどのシートも「← 戻る」で必ず元の文脈へ戻る。
4. 常設面(素材/文字起こし/タイムライン/音声)の入口が1つも減っていない。
5. ja/en を切り替えても全クロームが翻訳され、レイアウトが崩れない。
6. 意匠が d4 と一致(スクショ照合)し、意味色規律違反ゼロ。
7. e2e 全 green+reduced-motion で情報欠落なし。
