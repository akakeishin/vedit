# 到達可能性・操作可能性 総点検(2026-07-17、コードベース根拠)

3つの問題クラス: **機能はあるのに UI から到達できない / プレビューが最終出力を
保証しない / 操作したのに反応が見えない**。Sonnet 監査エージェントが
src/server/daemon.ts・src/cli.ts・src/export/render.ts・web/app.js ほかを
走査した結果の要約。行番号は調査時点(IA v2 波1 マージ前)のスナップショット。

対処状況の注記: ★=修正済み(2026-07-17中) / 波2=IA v2 波2 スコープ入り /
波3=要設計。

## 表3: 反応が見えない(要点)

| 操作 | 症状 | 対処 |
|---|---|---|
| 空タイムラインで再生 | 再生されないのに一時停止アイコン化(嘘) | 波2 |
| **クリップ境界の再生**(実測) | 0:01.4 で映像停止、UI は再生中のまま(状態不整合) | ★修正中(単独バッチ) |
| 編集成功時 | 成功フィードバック無し(エラー時トーストのみ) | 波2で方針決定 |
| 自動保存 | 「保存」という語が UI に一度も出ない | 波2(版表示に添える) |
| 保護区間の帯 | onclick 無し、カーソルも default = 壊れて見える | 波2 |
| オーファン B-roll の「!」 | トーストで説明だけして操作が無い | 波2/波3 |
