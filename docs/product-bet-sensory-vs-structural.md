# 製品の賭け: AIが先に働き、人間は例外と公開責任を持つ(2026-07-18)

旧テーゼ「感覚はUI、構造は会話」を改訂する。vedit の主語は NLE 操作ではなく
**AI編集者の仕事**である。ただし、人間の判断力を取り上げたり、根拠のない
自動化を「AIらしさ」と呼んだりはしない。

## 主ペルソナ

実ユーザー調査前の作業仮説として、主対象は週1〜3本のVlog・ショート・
解説動画を反復して公開する個人クリエイターと置く。

- 「もっとテンポよく」「この間は残したい」という作品判断はできる
- タイムライン操作の習得や、明白な無音を1件ずつ処理することは目的ではない
- AIに初稿まで任せたいが、自分の作風・説明可能性・公開責任は守りたい
- 必要なときは普通のNLEのように直接直せることも期待する

つまり、**判断力は高いが、反復操作への意欲は低い人**を中心に置く。
完全初心者向けの一発生成器でも、職業編集者向けの操作速度競争でもない。

## 自律度の原則

AIへの編集依頼が届いたら、確認待ちリストを作ること自体を成果にしない。
変更の性質に応じて次の4段階に分ける。

| 段階 | 条件 | 振る舞い |
|---|---|---|
| 自律実行 | 低リスク・可逆・機械可読な根拠あり | AIが先に1つのrevisionで実行し、件数・短縮秒数・根拠・Undoを報告 |
| 構造化質問 | 意味・好み、証拠不足/衝突、冒頭末尾の間、断片化で判断が割れる | 前後の文脈と再生入口を添え、「残す／カット」など答えられる形で聞く |
| 診断除外 | すでに保護/判断済み、不正区間、現在版に効果なし | 質問にも自律編集にも出さず、`aiReview`診断として記録 |
| 明示操作 | ローカルMP4生成や外部作用 | MP4はアプリ内ボタンで開始。外部送信・公開も将来は内容・宛先を示す別ボタンで明示確認 |

AI側から人間へ聞くインタラクティブ性そのものは制限しない。意味のある選択を
答えやすく聞けることも製品価値である。ただし、質問は安全儀式ではなく
**例外処理**である。判断根拠が足りない、文字起こしと
波形が衝突する、冒頭・末尾の間である、短い断片を巻き込む、フィラーを消すと
話者の印象が変わる、といった場合だけ人間へ戻す。すでにintent zoneとして
「残す」と決まっている間をもう一度質問しない。不正な候補、現在版に効果がない
候補、判断済み候補も、ユーザーの仕事ではなく診断除外である。

現在の自律初稿は特に保守的で、文字起こしの語間と波形の双方が一致した
内側の無音だけを対象にする。旧形式候補や片方だけの根拠は質問側へ倒す。
「音が静か」は「作品上不要」と同義ではないため、AIは素材構造を読み、
意味のある間を intent zone として先に保護する責任を持つ。保護後の候補は
黙って残し、必要なら診断で理由を追えるようにする。

## UIの役割

UIはAIの代わりに作業を人間へ返す場所ではなく、次を一目で理解できる場所にする。

- AIが何を、なぜ、どれだけ変えたか
- 本当に人間の判断が必要な箇所はどれか
- 自動適用も質問もされなかった候補を、必要時だけ診断として追えるか
- 同じ時刻を前後再生・タイムライン・文字起こしでどう確認するか
- 変更セットをどう比較・Undoするか
- どの版をローカルMP4へ書き出すか

タイムラインの直接操作は撤回しない。分割・トリム・移動・複製・ズーム・
ショートカット・Undo/Redoを商用NLE級へ近づける。これはAI主導と矛盾しない。
タイムラインは、AIの判断を検証し、例外を精密に直すための工具である。

アプリ内チャット入力は置かない。会話は Codex / Claude Code などの
AIエージェント側、実際の編集状態と証拠は vedit 側に置く。自律実行・
構造化質問・明示操作の境界は接続するAIプロバイダーで変えない。
ローカルMP4は「MP4をこのMacに書き出す」ボタンを正規入口とし、
プロジェクトの `exports/` に保存する。ボタンを押すまで処理は始めず、
外部へは送らない。YouTube等への送信・公開は別の外部作用であり、自動化しない。
将来追加する場合も、AIの自然文だけで開始せず、アプリ内の別ボタンに宛先・
アカウント・送信内容を表示して明示確認する。

## 合否

短期の機能仮説は次で測る。

- 明白な編集は1件ずつ質問せず、1つの変更セットとして適用される
- 60秒程度の動画で、人間への質問は意味・好みに関わる少数へ縮む
- 保護済み/不正/効果なし/判断済みの候補が質問数へ混ざらない
- 人間は10秒以内に「AIが何をしたか／何を聞いているか」を説明できる
- 自動変更は1操作で映像と候補状態をともにUndo/Redoできる
- UIを開いただけでは編集も書き出しも始まらない
- MP4ボタンは表示中のrevisionを固定し、二重実行・中断・失敗を扱える

この仮説の実装検証は [ai-first-validation-2026-07-18.md](ai-first-validation-2026-07-18.md)
に記録する。現時点で証明できたのは状態遷移・根拠ゲート・Undo/Redo・
実ffmpeg書き出しであり、実際のクリエイター素材で編集品質や質問数が
十分かは未証明である。

## 反転条件

- 根拠付き自動編集でも取り消し率が高い
- 意味のある間を継続的に壊す
- 質問が少なくても、理由が理解できず信頼されない
- 直接修正の方が常に速く、AIの初稿が作業短縮にならない
- 外部作用の境界が曖昧で、ユーザーが「勝手に送られた」と感じる

いずれかが実素材で反復するなら、自律範囲を狭める。逆に、シリーズ方針や
過去の明示判断から同じ選択が安定して再現できる場合だけ、自律範囲を段階的に
広げる。
