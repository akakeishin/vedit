# opencode UI根本再検討用プロンプト(コピーして新セッションへ)

---

あなたは vedit の **UI を根本から再検討する**独立デザイナー/レビュアーです。
現行 UI はこれまで Claude(Fable/Sonnet)が設計・実装してきました。あなたは
別のモデルとして、作った側の前提・構造・意匠に一切遠慮せず、**ゼロベースで
「この製品の UI はどうあるべきか」**を考え直してください。現行の批評だけでなく、
**あなた自身の UI 提案**(構造・画面・意匠)を出すことがゴールです。

## 対象

/Users/ht/dev/video-edit-skill — 会話型ローカル動画編集スキル。
Claude(Cowork/Code)が会話で編集を代行し、ユーザー(週次で作る個人
クリエイター)はブラウザ UI で確認・調整・直接編集する。実写 vlog/ショートと
「ゆる紙芝居」スプライトアニメの両方を扱う。

## 動かし方(必ず実物を操作すること)

- ビルド/テスト: `npm run build` / `npx vitest run` / `npm run test:e2e`
- アプリ起動: **port 7799 は使用禁止**(ユーザーの実 daemon が動いている
  可能性がある。絶対に kill しない)。自分用は
  `node dist/cli.js serve --port <空きポート> --project <dir>` で別途起動。
  検証用プロジェクトの作り方は e2e/fixtures.ts が完全な手本
  (隔離 HOME+ffmpeg lavfi 素材+合成 transcript)。
- Playwright で実クリック・スクリーンショットを撮り、**取り込み→カット→
  字幕→音量→書き出し確認の一連を通しで実操作**して、詰まり・迷い・
  冗長を全部記録すること。

## 再検討の問い(すべて「そもそも」から)

1. **メンタルモデル**: 「相棒が編集を進め、人は確認・調整する」体験の
   UI として、現行の 3カラム+タイムライン(調整室とキューシート)は
   正しい骨格か。まったく別の骨格(あなたの案)はあり得るか。
2. **商用比較**: CapCut / Premiere / DaVinci / Descript 等と並べたとき、
   初見の分かりやすさ・日常の使いやすさ・気持ちよさで決定的に負けている
   点はどこか。逆に会話型ならではの勝ち筋を UI はもっと活かせるか。
3. **意匠**: 「暗い計器の部屋+紙のキューシート」という視覚言語は
   実用に貢献しているか、雰囲気が実用を損ねている箇所はないか。
   フォントはシステムゴシック指定(ユーザー要件)。
4. **操作性**: NLE 級直接操作(分割/複数選択/ショートカット/undo 等、
   specs/2026-07-18-vedit-nle-operability.md)の計画は十分か。過剰か。
5. **一貫性と国際化**: 文言(日本語)の質、ja/en 切替の設計、
   アクセシビリティ(キーボード/コントラスト/reduced-motion)。

制約(これだけは前提として維持): ローカル完結/アプリ内にチャット入力を
置かない(依頼は Cowork 側)/書き出しの実行ボタンは置かない(結果表示のみ)。
これ以外の現行仕様・意匠・過去の却下判断は、すべて覆してよい。

## ルール

- リポジトリの既存ファイルは変更・コミットしない(read + run のみ)。
- **あなたの UI 提案は、一時ディレクトリ(例: /tmp/vedit-ui-rethink/)に
  静的 HTML モックアップ+スクリーンショットとして作ってよい**(推奨)。
  ワイヤーフレーム・構造図も歓迎。
- ユーザーの実データ(~/Movies 等)と port 7799 に触らない。
- 各指摘に: 重大度(P0=骨格を変えるべき/P1=近く直すべき/P2=改善)、
  根拠(実操作の再現手順 or file:line)、具体的な直し方。
- 儀礼的な賞賛・水増しは不要。ただし「変えるべきでない強み」も
  3点だけ明示すること(誤って壊さないため)。

## 出力

1. 総評(現行 UI の現在地を3行で)
2. 骨格への異議(あれば最優先 — 現行構造を維持すべきか、変えるなら何に)
3. **あなたの UI 提案**(構造・主要画面・意匠の方向。モックアップの
   パスとスクリーンショット添付)
4. P0 / P1 / P2 の指摘一覧(根拠・直し方つき)
5. 変えるべきでない強み3点
6. 実装順序の推奨(現行からの移行パス)
