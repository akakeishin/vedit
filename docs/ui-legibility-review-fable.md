# UI根本レビュー(Fable) — 「商用アプリ比でわかりにくい」の正体

日付: 2026-07-17。手法: 実プロジェクト(shibuya-final)を初見ユーザーとして
ブラウザで操作し、迷う瞬間を記録。比較対象は CapCut / Descript / Premiere /
Resolve の「誰でも迷わない」設計文法。

## テーゼ

**根本問題は「自己説明する文法」の不在。** 商用アプリは、どの領域も
「これは何か / ここで何ができるか / いま何が起きたか」を予備知識ゼロで
答える。現 UI は vedit の内部モデル(候補・keep/reject・revision・source)を
知っている人にしか読めない。色や角丸の問題ではない。

## 所見(重大度順)

### S1. 選択に反応がない(最重症)
タイムラインのクリップをクリックしても**見た目が何も変わらない**。
選択ハイライトなし、インスペクタ出現なし、次にできる操作の提示なし。
商用 NLE の中核文法「選択 → 右ペインが選択物のインスペクタになる」が欠落。
→ 選択ハイライト+右ペイン上部に選択インスペクタ(名前・範囲・
「この部分をどうしますか」相当のアクション)。

### S2. 「いま」タブの役割混在
タスク受信箱(要確認)/アルゴリズム設定(しきい値・再検出)/警告/
監査ログ(変更履歴)が同じ1カラムに同じ重さで積まれている。
商用文法ではタスク・設定・ログは絶対に同居しない。
→ 「提案」(バッジ付き)を独立させ、検出つまみは「再検出」の
オプション折りたたみへ、変更履歴は下部の控えめなセクションか
メニューへ格下げ。

### S3. トラックに名前がない
フィルムストリップ行・波形・B-roll・演出(モーション)・BGM の各行に
種別ラベルがない。初見は「この帯は何?」から始まる。
→ 左端に細いトラックラベルガター(映像 / 音声 / B-roll / 演出 / BGM)。
全商用 NLE 共通の文法。

### S4. 文言の英日混在・無説明の数値・凡例なし
- 候補タイトルが「0.6s silence after "かった」"」と英語テンプレート
- 「−0.3s」が何の数値か無ラベル(詰まる尺)
- Transcript の取り消し線(=カット済み)に凡例がない
→ 日本語テンプレート化(「"かった"の後の無音 0.6秒 — 詰めると −0.3秒」)、
状態の凡例をその場に。

### S5. 説明テキストの壁と崩壊
- Transcript タブ冒頭に操作説明の文字壁(改行が崩れる)
- 素材タブ右上のキーボード凡例が**数文字幅に潰れて判読不能**
→ パネル内の説明文を全廃し「?」オーバーレイに集約。操作は
アフォーダンス(ホバー時のヒント、コンテキストボタン)で教える。

### S6. 素材カードから素材の身元が消えている
サムネ+ボタン3つ(再生/タイムラインへ追加/詳細)だけで、
**名前・尺・使用状況がカード上にない**。
→ カード=サムネ+短い別名+尺+使用中バッジ。アクションはホバー/詳細へ。

### S7. 一等地の危険ボタンと重複
「すべて承認」が内容を見る前の最上部にあり、グループ側の「まとめて承認」と
重複。→ 一括操作はグループヘッダの1箇所に統一。

### S8. Claude 連携(製品の核)が不可視
候補は突然現れ、show カード(提案の提示)も出所の説明なくプレビューに
浮かぶ。「Claude が提案 → あなたが確認」という**この製品にしかない核**が
UI に語られていない。
→ 提案物に出所ラベル(「Claude の提案」)、候補カード ⇔ タイムライン上の
該当区間ハイライトの相互リンク(クリックで往復)。

### S9. Transcript が時間の地図になっていない
1ソース1段落の密なテキストで、タイムスタンプも段落分けもない。
Descript の文法は「時間に沿った段落+クリックでシーク」。
→ 発話塊ごとの段落+区間タイムコード。ファイル見出しの内部 ID
(srcmrnkyo9q1)は短い別名に。

## 実装への含意

S1〜S3 は情報設計の変更(パネル/タブ/トラック構造)、S4〜S7 は
コピー+レイアウトの修正、S8〜S9 は中規模の機能。Codex レビューと
突き合わせて「UI 情報設計 v2」仕様に統合し、1波で実装する。
