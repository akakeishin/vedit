# 編集判断プレイブック — 「いい感じに切って」を成立させる

曖昧・高レベルな指示(「いい感じに」「テンポよく」「ショートにして」)を
受けたときの段取り。機械的なカットツールではなく、編集判断こそが vedit の価値。

## 段取り(必ずこの順)

1. **分析**: transcript 分析エージェント(references/delegation.md テンプレート2)
   に素材の構造化を依頼。chapters / highlights / cutCandidates / pauses / risks。
2. **方針宣言**: 分析結果から編集方針を 2〜3 行でユーザーに見せる。必ず含める:
   - **約束**: この動画は視聴者に何を約束するか(冒頭で提示されるもの)
   - **見せ場**: どこがペイオフか(削ってはいけない山)
   - **余韻**: 終わり方(ぶつ切りにしない)
   例: 「『夜の渋谷を15分で回る』が約束。見せ場はスクランブル交差点の
   タイムラプスと屋台の会話。ラストの独白は余韻として無音ごと残します。」
3. **カットリスト提示**: 削る箇所を「引用+理由+秒数」の表で提示し、
   合計何秒→何秒になるかを言う。判断が割れる箇所(low confidence)は
   別枠にして UI の提案タブに積む(自分で決めない)。
4. **承認 → 適用**: 実行エージェント(テンプレート3)に委譲。
5. **確認**: view PNG とプレビュー URL をユーザーに提示。

## 「いい感じ」の判断材料

| 材料 | 見方 | 扱い |
|---|---|---|
| 冒頭フック | 最初の10秒に約束があるか | 無ければ後方から見せ場の一言を頭出しに移す提案 |
| テンポ | packed transcript の発話密度、(Ns pause) の頻度 | だれ場は圧縮。ただし↓ |
| **意味のある間** | 見せ場直後・感情の前後の pause | **切らない**。無音=悪ではない(Vrew への最大の不満は「感動の間まで切られる」) |
| 言い直し・重複 | ほぼ同じ内容の繰り返し | 良い方のテイクだけ残す(view で表情も確認) |
| 感情の山 | 笑い、驚き、声量(波形) | ハイライトとして保護 |
| 視覚的見せ場 | view PNG(風景、動きのあるフレーム) | 音声がなくても残す判断がある |
| 脱線 | 約束に寄与しない話題 | 尺目標次第で削る。丸ごと削る前にユーザーに引用を見せる |

**B-roll挿入の判断基準**: 話が続く区間の視覚的な単調さを補う。発話の意味と
無関係な良い画はショート素材に回す(`vedit broll-add` — アンカーは発話に
張り付くのでカット後も追従する)。

## transcript が信用できない素材(環境音の誤認識、"?" 多発)

夜の街歩きなど発話が少ない素材では、whisper が環境音を「会話」として
誤認識することがある。packed transcript の大半が意味不明・低confidence なら:

1. **字幕の無効化を提案する**(`vedit captions --enabled false`)—
   出典ルール上、誤認識テキストを字幕として焼くことはできない
2. 分析・編集の主軸を **scenes ワークフローに切り替える**
   (`vedit scenes detect` → `scenes sheet` → 視覚単位で構成判断)
3. カットリストの「引用」は view / scenes sheet の**セル参照**で代替する
   (例: 「cell9-12 の並木道ネオン区間」)

## 対抗案を出すときの規律(2軸ルール)

「テンポ重視」「丁寧め」のような複数案は、**最低2軸が変わっていなければ
別案と呼ばない**(例: 尺と構成順、フック位置と字幕密度)。語尾の違いだけの
水増し案を並べない。各案は「何を捨てるか」を明記する。

## 文言の出典ルール(捏造禁止)

- 字幕は transcript 由来のみ(自動生成)。**言っていないことを字幕にしない**
- チャプターカード・テロップの文言は (a) ユーザー提供、(b) transcript からの
  引用・要約、のどちらか。モデルの創作コピーは「提案」として提示し、
  ユーザー確認なしに焼かない
- 固有名詞は transcript の表記が信用できないことがある(whisper 誤認)。
  低confidence の固有名詞を字幕・カードに使う前に確認する

## レンダー前チェックリスト(全編書き出しの直前に必ず)

1. 約束: 冒頭10秒で動画の内容が分かるか
2. 尺: 目標に対して ±10% 以内か(`vedit status` の duration)
3. 字幕: `vedit captions`(引数なし)で cue 一覧を出し、重複・タイムライン超過・誤字がないか
4. カット境界: 各カット周辺の view で語尾切れ・ジャンプの違和感がないか
5. 終わり: 最後のクリップがぶつ切りになっていないか

## 部分プレビューの限界(ユーザーにも正直に)

`vedit view` と Web NLE のプレビューは編集判断には十分だが、
**全編の音量バランス・エンコード品質・字幕の最終見た目を保証しない**。
「公開できる」の判断は export render の実ファイルで行う。

## ショート切り出し(「このvlogからショート作って」)

1. 分析エージェントに「60秒で成立する自己完結の山」を3つ挙げさせる
2. ユーザーに選ばせる(引用+秒数で提示)
3. 選ばれた範囲だけの新プロジェクトを作り(元プロジェクトはそのまま)、
   冒頭に最強の一言をフックとして配置、字幕は大きめスタイル
4. `vedit reframe 9:16` + 縦向き export render

しゃべりのないB-roll区間を差し込みたい場合は `vedit scenes detect` +
`vedit scenes sheet` で視覚版の「packed transcript」を作り、注釈付きの
シーンを `clip-add --scene s0003` で直接指定する(タイムコード手計算不要)。

## 公開パックの起草手順

`vedit publish-pack <outdir>` は chapters.txt / thumbnails/ / materials.json
を生成するだけで、**タイトル・説明文・タグは書かない**(自動生成しない —
文言の出典ルールと同じ理由: モデル創作コピーは提案として見せ、承認後にだけ
書き込む)。起草はディレクター(あなた)が会話で行う:

1. `vedit publish-pack outdir` を実行し、materials.json と thumbnails/ を確認
   (thumbnails は Read してどれがサムネ候補として強いか目視判断する)
2. materials.json(duration / chapterList / sources / keptWordCount /
   captionsCueCount)と `vedit transcript` の引用を材料に、**タイトル案3つ・
   説明文1案・タグ案**をユーザーに提示する。説明文は transcript からの
   引用・要約かユーザー提供情報のみで組み立て、断定的な誇張(捏造を含む
   再生数・効果の主張等)を避ける
3. ユーザーが選択・修正して承認したら、その**承認済み文面をそのまま**
   `outdir/description.md` に書き込む(Write はディレクターが行う。
   モデルの下書きをユーザー承認なしに書き込まない)
4. chapters.txt はそのまま YouTube の概要欄チャプター欄に貼れる形式。
   thumbnails/ から実際に使うサムネをユーザーに選んでもらう
